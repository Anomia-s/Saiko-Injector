    2. Re-import LuduvoApp.exe into a fresh Ghidra project with:
       - "Windows x86 PE RTTI Analyzer" enabled

    4. For Qt-specific improvements, manually define key Qt types in the

       - QString (UTF-16 string with d-pointer)
       - QObject, QWidget, QMainWindow
       - QVector<T>, QList<T>, QMap<K,V>
       - QByteArray, QVariant
    5. Consider building Qt6 FID libraries using the Tools->Function ID
       dialog if any Qt code is statically linked

  HOW THE .exports FILES WORK:
    During analysis, Ghidra looks in <ghidra>/Ghidra/Features/Base/data/
    symbols/win64/ for files matching <dllname>.exports. When it encounters
    an import thunk to (e.g.) Qt6Core.dll, it resolves the ordinal or name
    from the PE import table against the .exports file to determine the
    correct mangled name. The mangled name is then demangled to produce a
    human-readable function signature that the decompiler can use.
